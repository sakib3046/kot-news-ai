TECHNICAL SUMMARY
──────────────────────────────────────────────────────────────────────────────

WHY THE BLACK SCREEN HAPPENED:
  1. globals.css file was corrupted (duplicated content)
  2. CSS parser failed on malformed file
  3. Dark mode detection: @media (prefers-color-scheme: dark)
  4. CSS rule applied: --background: #0a0a0a (pure black)
  5. Body element set to black background
  6. White login card became invisible on black
  7. Result: Complete black screen

WHY INLINE STYLES DIDN'T HELP:
  1. Global CSS is most powerful in cascade
  2. Body { background: ... } overrides component styles
  3. Dark mode CSS was too strong to override
  4. Cascade: Global CSS > component inline styles

THE FIX:
  1. Remove corrupted globals.css
  2. Create clean CSS with light mode forced
  3. Add: html { color-scheme: light; }
  4. This prevents dark mode CSS from triggering
  5. Page renders with proper light background
  6. All content becomes visible
